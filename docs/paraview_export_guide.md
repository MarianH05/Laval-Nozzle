# ParaView Export Guide

This guide describes how to create GitHub-ready screenshots from the OpenFOAM Laval nozzle cases. Do not commit placeholder or synthetic screenshots; only save images generated from actual case results.

## Open A Case

1. Ensure solved time directories already exist. If they do not, run the case intentionally after sourcing OpenFOAM:

   ```sh
   ./Allrun cases/choked
   ```

2. Open ParaView from the repository root:

   ```sh
   paraFoam -case cases/choked
   ```

   Alternatively, create an empty `.foam` marker and open it in ParaView:

   ```sh
   touch cases/choked/choked.foam
   paraview cases/choked/choked.foam
   ```

3. In ParaView, select the latest time, enable the internal mesh region, and click **Apply**.

4. Use **Surface** or **Surface With Edges** for contours. Use **Wireframe** or **Surface With Edges** for mesh screenshots.

## Calculate Mach If `Ma` Is Missing

If the `Ma` field is not available, create it in ParaView:

1. Select the case source in the Pipeline Browser.
2. Choose **Filters > Alphabetical > Calculator**.
3. Set **Result Array Name** to `Ma`.
4. Use this expression:

   ```text
   mag(U)/sqrt(1.4*287*T)
   ```

5. Click **Apply** and color by `Ma`.

## Recommended Camera Views

- Full nozzle contour: camera looking normal to the quasi-2D plane, with the full nozzle length visible.
- Mesh near throat: zoom around `x = 0.05 m`, centered on the minimum-area section.
- Shock view: for `cases/internal_shock`, zoom into the divergent section downstream of the throat where the Mach or pressure jump appears.
- Centerline plots: use the Python-generated plots from `docs/images/` rather than ParaView screenshots when possible.

Recommended display settings:

- Projection: parallel projection enabled.
- Background: white.
- Orientation axes: hidden for final screenshots unless useful.
- Color legend: enabled with readable numeric labels.
- Image width: at least 1600 px for contours.

## Export Steps

1. Set the desired field and camera view.
2. Choose **File > Save Screenshot**.
3. Save to the filename listed above.
4. Use PNG format.
5. Keep the image uncropped enough that the inlet, throat, outlet, and color legend are understandable.

After saving screenshots, verify the README image links render correctly on GitHub.
